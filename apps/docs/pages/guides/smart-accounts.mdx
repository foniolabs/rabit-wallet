# Smart accounts

Rabit ships with **ERC-4337 smart accounts** out of the box. That means gas sponsorship, batched transactions, and recoverable keys — without you wiring up bundlers, paymasters, or signers.

## Enable

Smart accounts are on by default. To opt out (use an EOA instead), pass `accountType="eoa"`:

```tsx
<RabitProvider apiKey={key} accountType="smart">
  {children}
</RabitProvider>
```

## Sponsor gas

```tsx
import { useSendTransaction } from '@rabit/react'

const { sendTransaction } = useSendTransaction({ sponsorGas: true })
```

When `sponsorGas` is true, the user signs the userOp but pays no gas — your project's paymaster pays.

## Batch transactions

```tsx
import { useSendCalls } from '@rabit/react'

const { sendCalls } = useSendCalls()

await sendCalls([
  { to: tokenAddress, data: approveData },
  { to: routerAddress, data: swapData },
])
```

Both calls land in a single userOp. One signature, one tx hash.
